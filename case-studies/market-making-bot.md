# Market-Making Bot Across 12 Exchanges

## Summary

Internal market-making automation built to manage order books and liquidity across 12 exchanges for KunciCoin's trading ecosystem.

## Role

Chief Technology Officer at KunciCoin. Designed and/or led implementation of internal automation for exchange order book and liquidity management.

## Key Capabilities

- Order book monitoring
- Liquidity management
- Exchange integration
- Automated market-making logic
- Internal trading ecosystem support

## Results

- Managed order books and liquidity across 12 exchanges.
- Strengthened KunciCoin's trading ecosystem and operational efficiency.

## Architecture Notes

A production-grade market-making system typically requires exchange API connectors, symbol/pair configuration, order book monitoring, spread and inventory rules, risk limits, dry-run/simulation modes, alerting, and operational controls. Public demo code should use mocked exchange connectors and no real API keys.

- Backend automation
- Exchange API integration
- Trading system architecture
- Reliability and operational thinking
- Ability to design internal tooling for business-critical workflows
